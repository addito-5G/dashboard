# Дашборд как продукт

Интерактивная карта продуктового подхода к дашбордам: 18 артефактов и 17 процессов с примерами для BI-систем (Power BI, Tableau, Looker).

**Сайт:** https://working-in-it.github.io/dashboard-as-a-product/

## Структура проекта

```
├── index.html           — главная страница (React-приложение)
├── data-artifacts.js    — данные: 18 артефактов (4 слоя)
├── data-processes.js    — данные: 17 процессов (6 фаз)
└── assets/              — иллюстрации к карточкам (~100 PNG)
```

## Как запустить локально

```bash
python3 -m http.server 8080
```

Открыть http://localhost:8080

## Как контрибьютить

### Редактирование текстов карточек

Тексты артефактов и процессов находятся в отдельных файлах — их удобно редактировать прямо на GitHub:

- **Артефакты** — [`data-artifacts.js`](data-artifacts.js)
- **Процессы** — [`data-processes.js`](data-processes.js)

Каждая карточка — объект с полями:

```js
{
  n: 1,                    // порядковый номер
  name: "Product Vision",  // название
  short: "...",            // текст на карточке (1-2 предложения)
  detail: `...`,           // слайд «Теория» (markdown)
  dashboard: `...`,        // слайд «Применение в BI» (markdown)
  bi: `...`,               // слайд «Функциональность BI» (markdown)
  detailImages: [...],     // картинки к слайду «Теория»
  dashboardImages: [...],  // картинки к слайду «Применение в BI»
  biImages: [...],         // картинки к слайду «Функциональность BI»
  maturity: "mvp",         // уровень зрелости: mvp | growth | mature
}
```

Текстовые поля (`detail`, `dashboard`, `bi`) поддерживают markdown:

- `### Заголовок` — заголовок секции
- `**жирный**` — выделение
- `[текст](url)` — ссылка
- `- пункт` — маркированный список
- `1. шаг` — нумерованный список

> Между заголовком `###` и предыдущим текстом нужен двойной перенос строки `\n\n`.
> Между пунктами списка `- ` — одинарный `\n`.

### Добавление и замена картинок

1. Положите PNG-файл в папку [`assets/`](assets/)
2. Именование: `{префикс}-{название}.png`
   - `a01-` ... `a18-` — для артефактов
   - `p01-` ... `p17-` — для процессов
   - `pbi-`, `tab-`, `look-` — скриншоты Power BI, Tableau, Looker
3. Добавьте ссылку в нужный массив картинок карточки:

```js
detailImages: [
  { src: "assets/a01-vision-template.png", caption: "Подпись под картинкой" }
],
```

Рекомендуемый размер: 1024px по ширине, формат PNG.

### Процесс внесения изменений

1. Форкните репозиторий
2. Внесите правки в `data-artifacts.js` или `data-processes.js`
3. Проверьте локально (`python3 -m http.server 8080`)
4. Создайте Pull Request с описанием изменений

### Что не нужно трогать

- `index.html` — в нём React-рантайм (инлайнён) и код компонентов. Редактировать только если меняете логику отображения.
